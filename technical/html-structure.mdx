## Overview

The portfolio landing page uses modern HTML5 semantic elements to create a well-structured, accessible, and SEO-friendly document. This page documents the structural patterns used across the three versions of the portfolio.

## Document Structure

### HTML5 Doctype and Language

All portfolio pages begin with the HTML5 doctype declaration and specify Spanish as the primary language:

```html
<!DOCTYPE html>
<html lang="es">
```

<Note>
Setting the `lang` attribute to "es" helps screen readers, search engines, and browsers properly interpret the content.
</Note>

### Meta Information

The `<head>` section includes essential meta tags for proper rendering and responsive design:

```html
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Luisa Fernanda Zuleta Hurtado</title>
</head>
```

**Key Meta Tags:**
- `charset="UTF-8"` - Ensures proper character encoding for Spanish characters and accents
- `viewport` - Enables responsive design by controlling page dimensions and scaling on mobile devices

## Content Sections

### Main Content Area

The portfolio uses the `<main>` element to wrap the primary content (in styled versions):

```html

    <img src="img/Fer.jpeg" alt="Luisa Fernanda Zuleta Hurtado" class="profile-img">
    
    <section id="perfil-profesional">
        <!-- Professional profile content -->
    </section>
```

## Accessibility Features

### Alt Text for Images

All images include descriptive alt text:

```html
<img src="img/Fer.jpeg" alt="Luisa Fernanda Zuleta Hurtado" class="profile-img">
<img src="img/facebook.png.png" alt="Facebook" width="30">
<img src="img/linkedin.png.png" alt="Linkedin" width="30">
```

### Semantic Heading Hierarchy

Proper heading structure from `<h1>` to `<h3>`:

```html
<h1>Luisa Fernanda Zuleta Hurtado</h1>           <!-- Page title -->
<h2>Perfil Profesional</h2>                       <!-- Section titles -->
<h3>Desarrollo de Software</h3>                   <!-- Subsection titles -->
```

### Link Accessibility

External links properly indicate they open in new windows:

```html
<a href="https://www.linkedin.com/..." target="_blank">
```

<Note>
Consider adding `rel="noopener noreferrer"` to external links with `target="_blank"` for security and performance.
</Note>

## Best Practices Implemented

✅ **HTML5 Semantic Elements** - Uses `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`  
✅ **Accessible Navigation** - Hash-based anchor links for smooth single-page navigation  
✅ **Responsive Meta Tag** - Viewport configuration for mobile devices  
✅ **Proper Character Encoding** - UTF-8 for international character support  
✅ **Alt Text on Images** - Descriptive text for screen readers  
✅ **Semantic Heading Hierarchy** - Logical progression of heading levels  

## Areas for Improvement

Consider these enhancements to the HTML structure:

1. **Add ARIA landmarks** for better screen reader navigation
2. **Include `rel="noopener noreferrer"`** on external links
3. **Restructure project list** to avoid mixing content types in `<ul>`
4. **Add meta description** for better SEO
5. **Use `<article>` tags** for self-contained content like project entries
6. **Add skip navigation link** for keyboard users
